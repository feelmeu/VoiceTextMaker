# ADR Format

仅当以下**三个条件同时成立**时才创建一个 ADR：

1. **难逆转** — 将来改变主意的代价显著
2. **没有上下文会让人困惑** — 未来的读者会问"他们为什么这么做？"
3. **是真实的权衡** — 有多个合理备选方案，你因具体理由选了其中一个

缺一个就不要创建 ADR。

---

## 文件名规范

`NNNN-短横线连接的标题.md`

- `NNNN` 是 4 位递增序号（从 `0001` 开始）
- 标题用英文小写、短横线连接

---

## 文件内容模板

```
# NNNN — 标题（一句话说明决策内容）

- 状态：Proposed / Accepted / Deprecated
- 日期：YYYY-MM-DD
- 替代：如果本 ADR 替代了之前的某个决策，在这里写明 `ADR-NNNN`

## 背景

描述当前面临的问题或需要决策的场景。

## 决策

我们决定……（一句话正面陈述决策内容）。

## 理由

- 理由 1
- 理由 2

## 备选方案与放弃原因

### 备选方案 A

描述。放弃原因：……。

## 后果

### 好处

-

### 风险/代价

-

## 不做的事

明确列出本次决策**不**包含的范围。
```

## 状态流转

| 状态 | 含义 |
|---|---|
| **Proposed** | 已写好，在讨论中，尚未执行 |
| **Accepted** | 已决定，正在或已经按此执行 |
| **Deprecated** | 该决策被新的 ADR 取代 |
