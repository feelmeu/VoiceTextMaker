# 0001 — 选择 GPT-SoVITS + MuseTalk 作为核心技术栈

- 状态：Accepted
- 日期：2026-06-24

## 背景

VoiceTextMaker 需要两个核心 AI 能力：语音克隆（从参考音频生成指定音色的语音）和唇形驱动（根据音频驱动视频中人物的嘴部动作）。市面上有多个成熟开源方案，需要做出选型决策。

## 决策

我们决定采用 **GPT-SoVITS**（语音克隆）+ **MuseTalk**（唇形驱动）的组合作为核心技术栈。

## 理由

**语音克隆 — GPT-SoVITS：**
- GitHub 5.79 万星，社区最活跃
- 支持零样本/少样本克隆（5秒参考音频即可）
- 中文效果在所有开源方案中最佳
- 有 Windows 整合包，降低部署难度
- 支持多语言（中/英/日/韩）

**唇形驱动 — MuseTalk：**
- 腾讯音乐 Lyra Lab 开源，工程质量高
- 实时性能好（V100 上 30fps+）
- 支持视频输入（不仅仅是图片）
- 潜空间 inpainting，画质损失小
- 2025 年开源训练代码，持续维护

## 备选方案与放弃原因

### Wav2Lip

精度高但生成画质一般（嘴部区域模糊），用户体验不佳。放弃原因：画质不满足"完整方案"的要求。

### SadTalker

支持图片+音频→视频，效果不错。放弃原因：唇形同步精度不如 MuseTalk，且对视频输入的支持不如 MuseTalk 完善。

### LatentSync（字节）

基于 Stable Diffusion 的唇形同步，效果好。放弃原因：项目较新，社区生态不如 MuseTalk 成熟，部署复杂度更高。

### CosyVoice / IndexTTS-2

同为优秀 TTS 方案。放弃原因：GPT-SoVITS 在中文场景和社区整合度上更优，且有现成 Windows 整合包。

## 后果

### 好处

- 使用最成熟的开源方案，效果有保障
- 社区活跃，遇到问题容易找到解决方案
- 两个项目都在持续维护更新

### 风险/代价

- 模型体积大（GPT-SoVITS ~2GB + MuseTalk ~1GB + 依赖），打包后 exe 可能较大
- 两个项目独立开发，集成时可能遇到接口不兼容
- 依赖 PyTorch + CUDA，GPU 兼容性需要测试

## 不做的事

- 不自研模型（用现有开源方案）
- 不做实时流式处理（首版为离线批量）
- 不做云端部署（首版为本地 Windows 程序）
